 Required next steps
──────────────────────────────────────────────────────────────────────────────

{{- if not .Values.serviceAccount.annotations }}

  ⚠ serviceAccount.annotations is empty. The operator pod needs an IRSA role
    to talk to AWS. Set the annotation:

      serviceAccount:
        annotations:
          eks.amazonaws.com/role-arn: arn:aws:iam::<acct>:role/<env>-<cluster>-eap-operator

    Read the role ARN from SSM (provisioned by terraform/components/agent-iam):

      aws ssm get-parameter \
        --name /eks-agent-platform/<env>/agent-iam/operator_role_arn \
        --query Parameter.Value --output text
{{- end }}

{{- if not .Values.config.environment }}

  ⚠ config.environment is empty. Set --set config.environment=dev|staging|production
    so the operator reads the right SSM tree.
{{- end }}
